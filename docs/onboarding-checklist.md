## Pre-Arrival (Felipe completes before Day 1)

- [ ] Google Chat Space invite sent
- [ ] User accepted invite and is visible in the Space
- [ ] EBFC AI container provisioned and running for this user
- [ ] Osito responds to a test message (send "ping" — confirm response)
- [ ] Quick-start guide shared in the Space via DM
- [ ] Welcome message sent from Felipe in the Space

**Welcome message template:**
> "Hey [Name] — welcome to the EBFC AI beta! You're one of our first 10 users and your feedback is going to shape how this product grows. Osito (your AI companion) is ready to go in this Space. Start by introducing yourself — just message Osito and tell it your name, role, and a project you're working on. The quick-start guide I sent covers the first steps. I'm here if you have questions."

---

## Day 1 Checklist

**Goal:** User has a profile set up, one project loaded, and has tried one real task.

- [ ] User sent their first message to Osito
- [ ] User completed profile setup (name, role, company)
- [ ] User loaded at least one project into Osito's memory
- [ ] User completed one real task (RFI draft, meeting minutes, daily report, or PPC)
- [ ] Felipe sent a check-in message: "How did the first session go? Anything feel clunky?"

**VOC questions for Day 1 check-in:**
1. What was your first impression?
2. Did the output from [task they tried] match what you'd actually use?
3. What would make it more useful right away?

---

## Day 3 Checklist

**Goal:** User has used at least 2 different skills and is building a habit.

- [ ] User has had at least 3 sessions with Osito
- [ ] User has tried at least 2 different skills (e.g., RFI + Meeting Minutes)
- [ ] Osito is referencing project memory correctly (test: ask "What's the status on [their project]?")
- [ ] Felipe sent a Day 3 check-in message

**Day 3 check-in message template:**
> "Hey [Name] — quick check-in on day 3. How's Osito working for you? Have you had a chance to try more than one skill? Anything it's getting wrong or that's frustrating? I want to hear the real feedback."

**VOC questions for Day 3:**
1. Which skill have you found most useful so far?
2. Is there something you tried to do and it didn't work the way you expected?
3. Are you using it daily, or does it feel like extra work to remember to use it?

---

## Day 7 Checklist

**Goal:** User has integrated EBFC AI into at least one regular workflow.

- [ ] User has been active at least 5 of the 7 days
- [ ] User has used 3+ skills
- [ ] User can describe a workflow where Osito saves them real time
- [ ] Memory is working — Osito references context from earlier in the week without being reminded
- [ ] Felipe scheduled a 15-minute call or async video check-in
- [ ] User has shared at least one piece of feedback (bug, suggestion, or compliment)

**Day 7 VOC questions (ask on call or via message):**
1. What's the one thing Osito does that you'd miss if it went away?
2. What does it still get wrong or where does it fall short?
3. Have you told anyone else about it? What did you say?
4. What would make it worth $49/month to you?

---

## Day 14 Retention Check

**Goal:** Confirm the user is coming back on their own — not just because we're watching.

- [ ] User has been active without prompting in the last 3 days
- [ ] User has a "go-to" use case (something they reach for Osito on without thinking)
- [ ] No unresolved technical issues (bot not responding, memory not working, etc.)
- [ ] User has been asked about sharing with a colleague
- [ ] NPS or satisfaction score collected (see below)

**Day 14 message template:**
> "Hey [Name] — you're two weeks in. Quick question: on a scale of 1-10, how likely are you to recommend EBFC AI to another PM or Super you know? And what's the one thing that would move that number up?"

**Day 14 VOC questions:**
1. What's your NPS score (1-10) and why?
2. Is there a workflow you wish Osito could handle that it can't yet?
3. Are you coming back every day? If not, what gets in the way?
4. What would you tell a skeptical colleague to convince them to try it?

## Escalation: When to Get Hands-On

If any of these are true at Day 7, schedule a live session with the user:

- They haven't used it more than 2-3 times
- They say "I keep forgetting about it"
- Osito gave them a bad output they haven't corrected
- They're having trouble getting the project loaded correctly

A 20-minute screen share where you walk through one real workflow together is worth more than any guide.
